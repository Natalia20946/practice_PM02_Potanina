ПК 2.4. Выполнять тестирование и отладку программных модулей.

ПК 2.5. Осуществлять администрирование баз данных (ведение, настройку, резервное копирование).

ПК 2.6. Применять методы и средства защиты информации в ИС.

2. Почему выбран стек: Python + MySQL + tkinter? Какие альтернативы могли быть?
Причины выбора:

Python: Простота синтаксиса, быстрое прототипирование, богатая стандартная библиотека и библиотеки для работы с БД.

MySQL: Надёжная реляционная СУБД, бесплатная (Community Edition), широко распространена, легко интегрируется с Python через mysql-connector-python или PyMySQL.

tkinter: Встроен в Python (не требует отдельной установки), достаточен для создания простого GUI под учебные задачи (CRUD-приложения).

Альтернативы:

Вместо Python: Java (Swing/JavaFX), C# (WinForms/WPF), PHP (для веб).

Вместо MySQL: PostgreSQL (мощнее), SQLite (легковесный, не требует сервера — идеален для локального прототипа), MS SQL Server.

Вместо tkinter: PyQt/PySide (профессиональнее, но сложнее установка), Kivy (под мобильные платформы), веб-фреймворки (Flask/Django + HTML/CSS для клиент-серверного приложения).

3. Что такое ER-диаграмма? Для чего она нужна?
ER-диаграмма (Entity-Relationship diagram) — это графическая нотация для описания логической структуры базы данных. Она показывает:

Сущности (таблицы) — прямоугольники.

Атрибуты (поля таблиц) — эллипсы/внутри таблиц.

Связи (отношения между таблицами) — ромбы/линии.

Нужна для:

Проектирования БД «на бумаге» до кода.

Согласования структуры данных между разработчиками и заказчиком.

Документирования существующей БД.

Легкой генерации SQL-скриптов из модели (в CASE-средствах).

4. Какие типы связей между таблицами существуют? Приведите примеры из своего варианта.
Существуют 3 основных типа связей (по кратности записей):

1:1 (один к одному): Одной записи в таблице А соответствует одна запись в таблице Б.

Пример: Пользователь ↔ Паспортные_данные (расширение таблицы).

1:M (один ко многим): Одной записи в А соответствует много записей в Б (самый частый).

Пример (из учебного варианта): Категория_товаров (1) → Товары (M).

M:N (многие ко многим): Требует промежуточной таблицы.

Пример: Студенты (M) ←→ Курсы (N) через таблицу Запись_на_курс.

5. Что такое Git? Чем отличается git commit от git push?
Git — распределённая система контроля версий, которая отслеживает изменения в файлах и позволяет нескольким разработчикам работать над одним проектом.

Отличия:

git commit — сохраняет изменения локально в репозитории на вашем компьютере (создаёт точку-слепок с сообщением).

git push — отправляет закоммиченные изменения из вашего локального репозитория в удалённый (например, на GitHub, GitLab).

Аналогия: commit — это «сохранить черновик на флешку», push — «отправить этот черновик в общее облако для коллег».

6. Как проверить, что Python установлен и tkinter работает?
Шаг 1 (проверка Python): Открыть терминал (cmd, PowerShell) и выполнить:

bash
python --version
# или
python3 --version
Должно вывести: Python 3.x.x

Шаг 2 (проверка tkinter): Запустить в терминале:

bash
python -m tkinter
Должно открыться маленькое тестовое окно Tk с кнопкой и заголовком.

Альтернативный скрипт:

python
import tkinter
tkinter._test()
7. Какие меры безопасности нужно соблюдать при работе за компьютером?
Электробезопасность: Не работать с мокрыми руками, не перегибать провода, использовать ИБП при скачках напряжения.

Санитарные нормы: Правильная осанка, расстояние до монитора 50–70 см, перерывы каждые 45–60 минут.

Информационная безопасность: Не использовать простые пароли, не переходить по подозрительным ссылкам, не вставлять чужие USB-накопители без проверки антивирусом, делать резервное копирование данных.

Пожарная безопасность: Не закрывать вентиляционные отверстия системного блока, не ставить технику рядом с нагревательными приборами.

8. Что означает IP-адрес 127.0.0.1?
127.0.0.1 — это локальный хост (localhost). Это стандартный IP-адрес, который всегда указывает на ваш же собственный компьютер. Сетевые запросы на этот адрес не выходят в реальную сеть (интернет/локальную), а обрабатываются внутри системы (loopback-интерфейс). Используется для тестирования серверов и сетевых приложений в одиночку.

9. Как добавить файл в репозиторий Git (последовательность команд)?
Предположим, вы в папке проекта и Git уже инициализирован (git init или клонирован).

Проверить статус (опционально):

bash
git status
Добавить файл в отслеживаемые (staging area):

bash
git add название_файла.py
# или все файлы: git add .
Зафиксировать изменения локально:

bash
git commit -m "Добавил новый файл с функцией X"
Отправить в удалённый репозиторий (если есть связь с сервером):

bash
git push origin main
# (или master, или имя_ветки)
10. Зачем нужен файл README.md в проекте?
README.md — это «лицо» проекта, файл с описанием, написанный в формате Markdown. Он нужен для:

Знакомства: Краткое название проекта и его цель.

Установки: Инструкция по развёртыванию (какие зависимости установить, как запустить БД).

Использования: Примеры кода, скриншоты GUI, описание API.

Документации: Кто авторы, какая лицензия, как связаться.

Автоматического отображения: GitHub/GitLab автоматически показывают его содержимое на главной странице репозитория. Без него пользователь видит только «голый» список файлов.
